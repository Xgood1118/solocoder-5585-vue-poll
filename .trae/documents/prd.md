## 1. 产品概述

社群运营纯前端投票系统——面向社群运营者的自托管投票工具，支持单选、多选、排序、评分四种投票类型，参与者扫码即投，发起人发链接即用，所有数据本地存储，无需后端服务。
- 解决第三方投票平台成本高、定制差、数据不可控的痛点
- 目标用户：社群运营人员、活动组织者、讲师，适用选题投票、报名投票、地点排序、讲师打分等场景

## 2. 核心功能

### 2.1 用户角色

| 角色 | 说明 | 核心权限 |
|------|------|----------|
| 发起人 | 创建投票的人 | 新建/编辑/复制/归档/删除投票、查看结果、导出数据、设置口令 |
| 参与者 | 通过链接/扫码加入的人 | 投票、查看实时结果、导出日历 |

### 2.2 功能模块

1. **投票列表页**：投票 CRUD、归档管理、搜索过滤、创建时间倒序排列
2. **投票创建/编辑页**：四种类型专属编辑器、选项图片/颜色配置、截止时间设置、口令设置
3. **投票参与页**：投票交互（单选/多选/排序/评分）、倒计时显示、时间校正
4. **实时结果页**：四种图表视图（柱状图/饼图/时间线/投票人列表）、选项明细隐藏/显示
5. **分享与导出**：一键复制链接、口令模式、JSON/ICS 导出

## 3. 核心流程

### 3.1 发起人创建投票流程
1. 发起人点击"新建投票"，填写标题和描述
2. 选择投票类型（单选/多选/排序/评分）
3. 使用对应类型编辑器添加选项，配置图片和颜色
4. 设置截止时间和可选口令
5. 点击发布，系统生成分享链接
6. 一键复制链接分享给参与者

### 3.2 参与者投票流程
1. 参与者通过链接/扫码进入投票页
2. 若有口令保护，先输入口令验证
3. 查看投票信息和截止倒计时
4. 根据投票类型进行投票交互
5. 提交投票，查看实时结果

### 3.3 实时结果更新流程
1. 参与者提交投票后，WebSocket 推送更新到所有在线参与者
2. 图表实时刷新，无需手动刷新页面
3. 断线后自动重连并恢复订阅

### 3.4 防刷机制流程
1. 参与者提交投票时，前端采集 Canvas 指纹、IP、User-Agent
2. 系统检查 IP 频次、UA 黑白名单、Canvas 指纹比对
3. 计算风险分，超过阈值触发熔断机制
4. 短时间内投票过快或指纹重复直接拒绝并提示

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：深海蓝 (#1a365d) + 活力橙 (#ff6b35) 点缀，沉稳专业又富有活力
- **辅助色**：浅灰背景 (#f7f8fa)、白色卡片、深灰文字 (#2d3748)
- **按钮风格**：圆角 8px，主按钮填充色，次按钮描边，悬浮微升阴影
- **字体**：标题用 Noto Sans SC Bold，正文用 Noto Sans SC Regular，数据数字用 JetBrains Mono
- **布局风格**：左侧导航 + 右侧内容区，卡片式布局，统一 16px 间距
- **图标**：使用 @iconify/vue 的 Material Design Icons 集合，线性风格
- **动效**：卡片悬浮微升、页面切换淡入、图表加载动画、拖拽排序过渡

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 投票列表页 | 操作栏 | 搜索框(左)、状态筛选标签(中)、新建按钮(右)，白色背景，底部 1px 分割线 |
| 投票列表页 | 投票卡片 | 白色圆角卡片，标题(18px bold)、类型标签(彩色小标签)、倒计时(橙色数字)、操作图标按钮，悬浮阴影加深 |

### 4.3 响应式设计
- 桌面优先设计，最小宽度 1024px
- 平板端（768-1024px）：卡片单列，侧边栏折叠为汉堡菜单
- 移动端（<768px）：全宽卡片，底部操作栏，触摸优化拖拽

### 4.4 暗色模式
- 支持亮色/暗色模式切换，跟随系统偏好或手动切换
- 暗色模式：深蓝背景 (#0f172a)、深灰卡片 (#1e293b)、浅色文字
